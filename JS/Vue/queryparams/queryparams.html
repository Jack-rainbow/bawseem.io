# query与params区别

1.传参
    路由中name必须与router文件中的name一样
        eg：
            router文件
            {
                path:"/xxx",
                name:"abc"
            }
            路由name
            this.$route.push({
                //切记是name不是path，params只能用name来引入路由
                name:"abc",
                params:{
                    name:'newabc',
                    code:10011
                }
            })
2.query类似于get传参(url中显示参数),params类似于post传参(url中不显示参数)
3.params、query不设置也可以传参，但是params不设置的时候
  刷新页面或者返回参数会丢失，query并不会出现这种情况
4.query获取的是参数名而params获取的是参数
    eg：
        /path:id 路由匹配/path/1这里的id用params
        /path?id=12  路由匹配/path?id=2这里的id用query
5.为什么刷新页面params会丢失参数？

6.使用路由参数(例如从 /user/foo 导航到 user/bar)后
  组件会被复用，此时应该使用$route在watch中进行监听
7.vue中前进后退
    router.go(1)  //进
    router.go(-1) //退
8.vue-router命名视图
    当页面需要使用多个router-view的时候,而不是嵌套展示.只需要设置name即可
    eg:
        <router-view class="view one"></router-view>
        <router-view class="view two" name="a"></router-view>
        <router-view class="view three" name="b"></router-view>
        const router = new VueRouter({
          routes: [
            {
              path: '/',
              components: {
                default: Foo,
                a: Bar,
                b: Baz
              }
            }
          ]
        })
