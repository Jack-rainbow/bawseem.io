# vuex

🐶

什么时候用vuex？

1.vue中父给孙子传值的时候可能需要用到vuex，因为在vue中没有父给孙子传值这个说法，要么借助bus总线，要么vuex

2.数据公用，不同组件都需要得到这个数据

3.多个组件之间互相传值或者修改值

vuex有3个状态

Actions ---- 一般用于异步操作

Mutations ---- vuex中的主要修改数据的地方

State ---- 存储着vuex中的状态值

vuex一般分为6个文件

1.index.js

index.js主要存储一些配置文件及加载其他5个js文件

2.state.js

state是用来储存状态值的
eg:
    const state = {
        city: {},
    }
    export default state;

3.getter.js

用来获取state中的值且不做任何修改及对数据进行一些操(过滤,修改)
eg:
    cosnt city = state => state.city

4.mutations_types.js

用来存储mutations中一些常量
eg:
    expport const SET_CITY = 'SET_CITY'

5.mutations.js

获取mutations中的一些常量及获取修改state的值。且接受commit(组件和Actions传过来)的值,
mutations必须是同步函数
eg:
    import * as types from './mutations_types.js'
    export const mutations ={
        //常量函数名
        [types.SET_CITY] (state,city){
            //state是状态值
            //city是参数值
            state.city =city
        }
    }
    export default mutations;

6.actions

用来一些异步操作(Dispatch传来)的值且commit可以传送到mutations中
eg:
    import * as types from './mutations_types.js'
    export const changcity = function({commit},data){
        commit(types.SET_CITY,data)
    }

组件内调用使用 ...mapActions/mapmutations/mapState/mapGetters

1.组件中直接使用
    this.$stroe.getters/state/mutations.changcity
2.computed:{
    ..mapActions([
        //Actions中的文件名
        //映射this.changcity 为store.state.changcity
        'changcity'
    ])
}
3.组件同步提交值--mutations
    this.$store.commit('xxx')
4.组件异步提交值--Actions
    this.$store.dispatch('xxx')
